This web application shows different visualizations of stackowerflow web-sites. Source data for processing taken here. For data processing used machine learning clustering algorithms. Calculations done using by Mahout library, Apache Hadoop and data stored in Neo4j graph database.

Research work based on this code, by Frank Scholten.

Visualizations photo.stackexchange.com

Search clusters and posts (photo)

Posts clustered using K-Means algorithm. This demo shows list of clusters (posts groups). You view list of clusters, search them and view posts belongs to each cluster.

Dataset: full photo.stackexchange.com

Algorithm: K-Means clustering

Clusters count: constant 250

Tools and technologies used

Apache Hadoop

Apache Hadoop used as the basis for all calculations and for pre-processing and post-processing data. Before run any machine learning algorithm we beed to do several preparation steps like removing stop-words, extracting data from raw source files, preparing input data for mahout machine learning library (save in specific format).

Go to website »

Apache Mahout

Library with machine-learning algorithms over hadoop. Contains a lot of ready map-reduce jobs for clustering, classification and recomendation systems development. For processing stackexchange data used TF-IDF, K-Means, Fuzzy K-Means clustering algorithms.

Go to website »

Neo4j graph database

Graph database used for storing custered data and relationships. Graph database is good as related objects storage.
